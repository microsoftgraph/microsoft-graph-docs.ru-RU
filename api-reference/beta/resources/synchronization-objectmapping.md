# <a name="objectmapping-resource-type"></a>Тип ресурса objectMapping

Пространство имен: microsoft.graph

[!INCLUDE [beta-disclaimer](../../includes/beta-disclaimer.md)]

Определяет, как следует синхронизировать заданный объект из каталога источника в целевой каталог. В частности, он определяет, как объект в каталоге источника должен быть совмедан с объектом в целевом каталоге, какие (при их наряду) фильтры области следует использовать для временного решения о том, нужно ли нам подготовка конкретного объекта, и как следует преобразовывать атрибуты объекта из источника в целевой каталог.

Сопоставления объектов являются основной [](synchronization-synchronizationrule.md) частью правила синхронизации и обновляются в рамках [схемы синхронизации.](synchronization-synchronizationschema.md)

## <a name="properties"></a>Свойства

| Свойство      | Тип      | Описание    |
|:--------------|:----------|:---------------|
|attributeMappings  |[Коллекция attributeMapping](synchronization-attributemapping.md)    | Сопоставления атрибутов определяют, какие атрибуты соотнося из объекта-источника в целевой объект и как они должны поступать. Для поддержки преобразования исходных исходных значений доступно несколько функций.|
|enabled        |Boolean    |Когда `true` это сопоставление объектов будет обработано во время синхронизации. Когда `false` это сопоставление объектов будет пропущено.|
|flowTypes      |objectFlowTypes    |Типы потоков, которые включены для этого сопоставления объектов. `Add` создает новые объекты в целевом каталоге, изменяет существующие объекты `Update` и отгвоирует `Delete` существующих пользователей. Значение по `Add, Update, Delete` умолчанию: . |
|метаданные       |Коллекция metadataEntry    |Дополнительные свойства расширения. Если не было явно упомянуто, значения метаданных не следует менять.|
|name           |String     |Удобное для человека имя сопоставления объектов.|
|scope          |[filter](synchronization-filter.md)     |Определяет фильтр, который будет использоваться при принятии решения о том, следует ли подготовка данного объекта. Например, может потребоваться только подготовка пользователей, расположенных в США.|
|sourceObjectName           |Строка     |Имя объекта в каталоге источника. Должен соответствовать имени объекта из определения [источника каталога.](synchronization-directorydefinition.md)|
|targetObjectName           |Строка     |Имя объекта в целевом каталоге. Должен соответствовать имени объекта из определения [целевого каталога.](synchronization-directorydefinition.md)|
