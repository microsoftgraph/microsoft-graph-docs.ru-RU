# <a name="partnering-with-the-microsoft-graph-security-api--technology-partner-opportunities"></a>Партнерство с API безопасности microsoft Graph — возможности партнеров по технологиям

В этой статье описываются возможности партнерских партнеров, включенные API microsoft Graph безопасности, и она предназначена для того, чтобы помочь руководителям продуктов и ролям по развитию бизнеса понять пути инвестиций и получить представление о предложениях по добавленной стоимости для партнеров.

## <a name="background"></a>Общие сведения

Большинство организаций имеют дело с высокими объемами данных безопасности и имеют десятки решений по обеспечению безопасности в своем предприятии, что делает задачу интеграции различных продуктов и служб сложной и сложной. Эти проблемы препятствуют быстрому перемещению организаций при обнаружении и устранении угроз в мире быстрых и разрушительных атак.

Технологические партнеры могут интегрироваться с платформой Майкрософт с помощью API Graph безопасности Майкрософт для решения этих проблем клиента.

## <a name="introduction-to-the-microsoft-graph-security-api"></a>Введение в API безопасности Graph Майкрософт

Microsoft Graph Security API — это единый API, который предоставляет стандартный интерфейс и единую схему для интеграции оповещений от системы безопасности и аналитических данных об угрозах из нескольких источников, обогащает оповещения и данные контекстной информацией, а также автоматизирует операции системы безопасности.

API безопасности входит в состав Microsoft Graph — единого REST API для интеграции данных и аналитики от Майкрософт, а также партнерских продуктов и служб. Благодаря Microsoft Graph клиенты и партнеры могут быстро создавать решения, которые проходят проверку подлинности один раз и используют один вызов API для доступа к аналитическим сведениям о системе безопасности из различных решений и принятия соответствующих мер. Дополнительные значения раскрывают при изучении других сущно-Graph [Майкрософт](./overview.md) (Microsoft 365, Azure Active Directory, Intune и других), чтобы связать бизнес-контекст с вашими сведениями о безопасности.

Корпорация Майкрософт включает интеграцию технологических партнеров двумя ключевыми способами.

1. Как потребитель информации из Microsoft Graph, вы можете обогатить решения информацией, содержаной в Microsoft Graph, а также использовать API Microsoft Graph для выполнения задач от имени клиента.
2. Вы также можете вносить оповещения и действия в корпорацию Майкрософт Graph наряду с поставщиками Майкрософт.

Давайте углубемся и рассмотрим некоторые распространенные сценарии, в которых интеграция microsoft Graph API безопасности увеличивает инвестиции в интеграцию безопасности и преимущества для клиентов, которых мы можем достичь вместе.

## <a name="featured-technology-partner-scenarios"></a>Сценарии партнеров по рекомендуемой технологии

Ниже 3 ключевых преимущества, которые можно извлечь, интегрируясь с API Graph безопасности Майкрософт:

1. Ваши клиенты получают выгоду от повышения эффективности безопасности и операций.
2. Ваши клиенты получают богатые сведения, предоставленные вашими и другими интегрированными партнерскими продуктами.
3. Инвестиции в разработку для технологических партнеров упрощаются, а клиентская ценность увеличивается с помощью интеграции с API Graph безопасности Майкрософт.

### <a name="enhance-threat-protection-with-the-microsoft-graph-security-api"></a>Повышение защиты от угроз с помощью API Graph безопасности Майкрософт

*Включение более простой интеграции оповещений о безопасности для информирования об обнаружении угрозы и реагировании на них.*

- Сопоставление оповещений и обнаружения от поставщиков Graph безопасности с обнаружением для улучшения результатов расследования и поддержки автоматизации.
- Обнаружение и контекст доступа через службу майкрософт Graph для улучшения реагирования на угрозы — триадж, исследование, исправление.
- Доступ к сведениям об угрозах клиентов (hash, IP, URL-адрес, домен и т.д.) для блокировки и оповещения о вредоносных действиях.

### <a name="streamline-it-and-security-management"></a>Оптимизация управления ИТ-службами и безопасностью

*Обеспечение большей видимости и оптимизация управления жизненным циклом инцидента.*

- Сводка оповещений от нескольких поставщиков для создания инцидентов.
- Доступ к дополнительному контексту для информирования о приоритетах и ответах оповещений.
- Синхронизуй состояние оповещений в системах управления оповещениями.
- Получение видимости в области безопасности и рекомендации по ее улучшению с помощью secure Scores.

### <a name="share-threat-intelligence-to-enable-custom-detections"></a>Share threat intelligence to enable custom detections

*Использование сведений об угрозах для использования настраиваемой системы обнаружения в решениях Майкрософт.*

- Автоматически отправляйте индикаторы угроз в решения по безопасности Майкрософт, чтобы включить действия Alert, Block или Allow.
- Включить быстрые действия для защиты от новых угроз, таких как блок-файл, URL-адрес, домен, IP-адрес из средств безопасности и рабочего процесса.
- Предоставленный клиентом TI используется только для клиента-поставок, а не для любого другого клиента Майкрософт.

## <a name="technical-integrations-overview"></a>Обзор технических интеграций

Возможности Graph API безопасности доступны через два основных пути интеграции, которые можно использовать независимо или совместно.  Мы нарисуем требования высокого уровня и предозначим некоторые сведения о том, как думать об инвестициях в эти пути, но подробные технические пояснения слева на документах, указанных ниже в этом документе.

Поддерживаемые сущности:

- **Оповещений** являются "выводы с влиянием на безопасность", а не необработанные данные журнала или другие некорректные сведения. [Подробнее](/graph/api/resources/security-api-overview#alerts).
- **Индикаторы** угроз, также именуемые индикаторами компромисса или IoCs, представляют данные об известных угрозах, таких как вредоносные файлы, URL-адреса, домены и IP-адреса. Клиенты могут создавать индикаторы с помощью внутреннего сбора сведений об угрозах или получать индикаторы из сообществ сведении об угрозах, лицензированных каналов и других источников. [Подробнее](/graph/api/resources/tiindicator).
- **Действия безопасности** позволяют технологическим партнерам раскрыть функциональные возможности с помощью Graph.  Например, если решение безопасности поддерживает возможность блокировки IP-адресов, вы можете выставить "Block IP" как возможность в Graph. Другие Graph API безопасности могут вызывать ваше действие через Graph. [Подробнее](/graph/api/resources/securityaction).
- **Безопасная оценка**... [Подробнее](/graph/api/resources/securescores).

### <a name="integrate-your-application-with-the-microsoft-graph-security-api"></a>Интеграция приложения с API Graph безопасности Майкрософт

Все интегрированные приложения [должны быть зарегистрированы](./auth-register-app-v2.md) в Microsoft Graph. Поддерживаются как приложения, используемые одним клиентом, так и приложения, используемые многими клиентами (несколькими клиентами).  В любом случае клиент должен предоставить согласие на ваше приложение. При вызове Graph Майкрософт каждый запрос из приложения будет содержать идентификатор приложения и клиента, от имени которых вы вызываете. Поддерживаются следующие типы запросов:

- **Получить оповещений** — получить сведения об оповещении с помощью фильтрации по мере необходимости.  Например: покажите мне все оповещений с высоким приоритетом или "все оповещений с высоким приоритетом" для конкретного пользователя, хоста и т.д.
- **Обновление состояния оповещений** — включение управления жизненным циклом оповещений.  Например: настройка состояния оповещений на "разрешено" из "в процессе" или добавление комментариев к оповещению.
- **Получить безопасную** оценку — microsoft Secure Score — это значение типа "кредитная оценка" для конфигураций безопасности продуктов Майкрософт.
- **Подписка** — разрешение уведомления об изменениях оповещений или запросов.
- **Подача настраиваемого** индикатора угрозы — автоматически отправлять индикаторы угроз в решения безопасности Майкрософт, чтобы включить действия Alert, Block или Allow. Используйте API Graph безопасности майкрософт напрямую или используйте интеграцию с ведущими платформами разведки угроз.
- **Вызов меры безопасности Graph** Майкрософт — немедленно принять меры для защиты от угроз с помощью объекта безопасности microsoft Graph [securityActions.](/graph/api/resources/securityaction)

### <a name="enable-others-to-integrate-with-your-products-through-the-microsoft-graph-security-api"></a>Включить интеграцию других с вашими продуктами через API Graph безопасности Майкрософт

Поставщики Graph майкрософт делают свои оповещения о безопасности доступными для других через microsoft Graph.  Продукты Корпорации Майкрософт, которые генерируют оповещения о безопасности, имеют поставщиков, которые выставляют соответствующие оповещения Graph. Кроме того, API Graph безопасности Майкрософт позволяет внешним поставщикам, как технологическому партнеру Майкрософт, обмениваться соответствующими оповещениями о безопасности из приложений в microsoft Graph для пользователей. Помимо оповещений microsoft Graph securityActions позволяет технологическим партнерам раскрыть функциональные возможности с помощью microsoft Graph.  Например, если решение безопасности поддерживает возможность блокировки IP-адресов, вы можете выставить "Block IP" в качестве возможности в microsoft Graph. Другие продукты майкрософт Graph безопасности могут вызвать ваше действие через microsoft Graph.

Поставщик безопасности Graph Майкрософт по сути является облачной конечной точкой, которая отвечает на запросы API безопасности Microsoft Graph и возвращает соответствующие оповещения о безопасности или выполняет действия для общих клиентов. Проверка подлинности между клиентами и службами обеспечивает доступ к оповещениям и действиям клиентов.

Сценарии поставщика очень разнообразны. Кураторский процесс бортовой обработки начинается с определения соответствующих сценариев. После согласований сценариев документация, пример кода и среды разработки будут доступны для поддержки разработки поставщика Graph microsoft.
