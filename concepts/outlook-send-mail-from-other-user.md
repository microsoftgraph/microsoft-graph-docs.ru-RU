<span data-ttu-id="1c6c8-109">Чтобы отправлять сообщения от другого пользователя, приложения, использующие маркеры пользователя, используют разрешение **Mail.Send.Shared**.</span><span class="sxs-lookup"><span data-stu-id="1c6c8-109">In order to send messages from another user, applications that use user tokens use the **Mail.Send.Shared** permission.</span></span>

### <a name="mailbox-permissions"></a><span data-ttu-id="1c6c8-111">Разрешения для почтового ящика</span><span class="sxs-lookup"><span data-stu-id="1c6c8-111">Mailbox permissions</span></span>

#### <a name="send-on-behalf"></a><span data-ttu-id="1c6c8-114">Отправить от имени</span><span class="sxs-lookup"><span data-stu-id="1c6c8-114">Send on Behalf</span></span>

<span data-ttu-id="1c6c8-115">С таким разрешением получатель почты получает указание в почтовом клиенте, что сообщение было отправлено пользователем приложения от имени другого пользователя.</span><span class="sxs-lookup"><span data-stu-id="1c6c8-115">With this permission, the recipient of the email has an indication in their email client that the message was sent by the user of your application on behalf of another user.</span></span>

![Снимок экрана Outlook в Интернете с указанием того, что сообщение было отправлено одним пользователям от имени другого](images/outlook-sent-on-behalf.png)

<span data-ttu-id="1c6c8-117">В Microsoft Graph это представлено в виде свойств `sender` (пользователь, который фактически отправил сообщение) и `from` (пользователь, группа или другое, отображаемые в качестве отправителя).</span><span class="sxs-lookup"><span data-stu-id="1c6c8-117">This is exposed in Microsoft Graph as the `sender` (user that actually sent the message) and `from` (user/group/etc. that the message appears to be from) properties.</span></span>

```json
{
  "id": "AAMkAGE1...",
  "subject": "Send mail test",
  "sender": {
    "emailAddress": {
      "name": "Adele Vance",
      "address": "AdeleV@contoso.com"
    }
  },
  "from": {
    "emailAddress": {
      "name": "Pradeep Gupta",
      "address": "PradeepG@contoso.com"
    }
  }
}
```

<span data-ttu-id="1c6c8-118">Пользователь может предоставлять это разрешение для своего почтового ящика другому пользователю, [используя Outlook](https://support.office.com/article/Allow-someone-else-to-manage-your-mail-and-calendar-41C40C04-3BD1-4D22-963A-28EAFEC25926).</span><span class="sxs-lookup"><span data-stu-id="1c6c8-118">A user can grant this permission for their own mailbox to another user by [using Outlook](https://support.office.com/article/Allow-someone-else-to-manage-your-mail-and-calendar-41C40C04-3BD1-4D22-963A-28EAFEC25926).</span></span> <span data-ttu-id="1c6c8-119">Администраторы могут предоставить это разрешение для любого почтового ящика, группы или списка рассылки в [Центре администрирования Microsoft 365](/office365/admin/add-users/give-mailbox-permissions-to-another-user?view=o365-worldwide).</span><span class="sxs-lookup"><span data-stu-id="1c6c8-119">Administrators can grant this permission for any mailbox, group, or distribution list in the [Microsoft 365 admin center](/office365/admin/add-users/give-mailbox-permissions-to-another-user?view=o365-worldwide).</span></span>
